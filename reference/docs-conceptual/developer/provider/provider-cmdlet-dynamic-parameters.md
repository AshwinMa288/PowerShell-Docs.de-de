# <a name="provider-cmdlet-dynamic-parameters"></a><span data-ttu-id="52179-102">Dynamische Anbieter-Cmdlet-Parameter</span><span class="sxs-lookup"><span data-stu-id="52179-102">Provider cmdlet dynamic parameters</span></span>

<span data-ttu-id="52179-103">Anbieter können dynamische Parameter definieren, die einem Anbieter-Cmdlet hinzugefügt werden, wenn der Benutzer einen bestimmten Wert für einen der statischen Parameter des Cmdlets angibt.</span><span class="sxs-lookup"><span data-stu-id="52179-103">Providers can define dynamic parameters that are added to a provider cmdlet when the user specifies a certain value for one of the static parameters of the cmdlet.</span></span> <span data-ttu-id="52179-104">Ein Anbieter kann z. b. verschiedene dynamische Parameter basierend auf dem Pfad hinzufügen, den der Benutzer beim Abrufen der Cmdlets für `Get-Item` oder `Set-Item` Anbieters angibt.</span><span class="sxs-lookup"><span data-stu-id="52179-104">For example, a provider can add different dynamic parameters based on what path the user specifies when they call the `Get-Item` or `Set-Item` provider cmdlets.</span></span>

<span data-ttu-id="52179-106">Dynamische Parameter werden definiert, indem eine der dynamischen Parameter Methoden implementiert wird, wie z. b. die [System. Management. Automation. Provider. itemcmdletprovider. getitemdynamicparameters \*](/dotnet/api/System.Management.Automation.Provider.ItemCmdletProvider.GetItemDynamicParameters) -und [System. Management. Automation. Management. Automation. Provider. itemcmdletprovider. getitemdynamicparameters \*](/dotnet/api/System.Management.Automation.Provider.ItemCmdletProvider.GetItemDynamicParameters) -Methode, die vom Zertifikat Anbieter stammt:</span><span class="sxs-lookup"><span data-stu-id="52179-108">Here is an example of an implementation of the [System.Management.Automation.Provider.Itemcmdletprovider.Getitemdynamicparameters\*](/dotnet/api/System.Management.Automation.Provider.ItemCmdletProvider.GetItemDynamicParameters) method taken from the Certificate provider:</span></span>

```csharp
protected override object GetItemDynamicParameters(string path)
{
    return new CertificateProviderDynamicParameters();
}
```

<span data-ttu-id="52179-109">Im Gegensatz zu den statischen Parametern von Provider-Cmdlets können Sie die Eigenschaften dieser Parameter auf die gleiche Weise angeben, wie Parameter in eigenständigen Cmdlets definiert werden.</span><span class="sxs-lookup"><span data-stu-id="52179-109">Unlike the static parameters of provider cmdlets, you can specify the characteristics of these parameters in the same way that parameters are defined in stand-alone cmdlets.</span></span> <span data-ttu-id="52179-110">Im folgenden finden Sie ein Beispiel für eine dynamische Parameter Klasse, die vom Zertifikat Anbieter übernommen wird:</span><span class="sxs-lookup"><span data-stu-id="52179-110">Here is an example of a dynamic parameter class taken from the Certificate provider:</span></span>

```csharp
internal sealed class CertificateProviderDynamicParameters
{
  /// <summary>
  /// Dynamic parameter the controls whether we only return
  /// code signing certs.
  /// </summary>
  [Parameter()]
  public SwitchParameter CodeSigningCert
  {
    get
    {
      {
        return codeSigningCert;
      }
    }

    set
    {
      {
        codeSigningCert = value;
      }
    }
  }

    private SwitchParameter codeSigningCert = new SwitchParameter();
}
```
