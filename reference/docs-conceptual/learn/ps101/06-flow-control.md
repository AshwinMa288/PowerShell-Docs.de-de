<span data-ttu-id="66114-115">Obwohl der Parameter **Module** von `Get-Command` mehrere Werte akzeptiert, die Zeichenfolgen sind, akzeptiert er sie nur über die Pipelineeingabe mittels Eigenschaftsname oder über Parametereingaben.</span><span class="sxs-lookup"><span data-stu-id="66114-115">Although the **Module** parameter of `Get-Command` accepts multiple values that are strings, it only accepts them via pipeline input by property name or via parameter input.</span></span> <span data-ttu-id="66114-116">Wenn Sie im folgenden Szenario zwei Zeichenfolgen als Wert an `Get-Command` übergeben möchten, um sie mit dem Parameter **Module** zu verwenden, müssten Sie das Cmdlet `ForEach-Object` verwenden.</span><span class="sxs-lookup"><span data-stu-id="66114-116">In the following scenario, if I want to pipe two strings by value to `Get-Command` for use with the **Module** parameter, I would need to use the `ForEach-Object` cmdlet.</span></span>

### <a name="for"></a><span data-ttu-id="66114-126">Für</span><span class="sxs-lookup"><span data-stu-id="66114-126">For</span></span>

<span data-ttu-id="66114-127">Eine `for`-Schleife wird durchlaufen, während eine bestimmte Bedingung erfüllt (true) ist.</span><span class="sxs-lookup"><span data-stu-id="66114-127">A `for` loop iterates while a specified condition is true.</span></span> <span data-ttu-id="66114-128">Die `for`-Schleife verwende ich nicht häufig, aber sie hat ihre Berechtigung.</span><span class="sxs-lookup"><span data-stu-id="66114-128">The `for` loop is not something that I use often, but it does have its uses.</span></span>

```powershell
}
```

```Output
Sleeping for 1 seconds
Sleeping for 2 seconds
Sleeping for 3 seconds
Sleeping for 4 seconds
```

<span data-ttu-id="66114-129">Im vorherigen Beispiel wird die Schleife viermal durchlaufen, beginnend mit der Zahl 1, und wird fortgesetzt, solange die Zählervariable `$i` kleiner als 5 ist.</span><span class="sxs-lookup"><span data-stu-id="66114-129">In the previous example, the loop will iterate four times by starting off with the number one and continue as long as the counter variable `$i` is less than 5.</span></span> <span data-ttu-id="66114-130">Sie setzt für insgesamt 10 Sekunden aus.</span><span class="sxs-lookup"><span data-stu-id="66114-130">It will sleep for a total of 10 seconds.</span></span>

### <a name="do"></a><span data-ttu-id="66114-131">Sie sollten</span><span class="sxs-lookup"><span data-stu-id="66114-131">Do</span></span>

<span data-ttu-id="66114-132">Es gibt zwei verschiedene `do`-Schleifen in PowerShell.</span><span class="sxs-lookup"><span data-stu-id="66114-132">There are two different `do` loops in PowerShell.</span></span> <span data-ttu-id="66114-133">`Do Until` wird ausgeführt, während die angegebene Bedingung nicht erfüllt (false) ist.</span><span class="sxs-lookup"><span data-stu-id="66114-133">`Do Until` runs while the specified condition is false.</span></span>

```powershell
$number = Get-Random -Minimum 1 -Maximum 10
do {
  $guess = Read-Host -Prompt "What's your guess?"
  if ($guess -lt $number) {
    Write-Output 'Too low!'
  }
  elseif ($guess -gt $number) {
    Write-Output 'Too high!'
  }
}
until ($guess -eq $number)
```

```Output
What's your guess?: 1
Too low!
What's your guess?: 2
Too low!
What's your guess?: 3
```

<span data-ttu-id="66114-134">Das vorherige Beispiel ist ein Zahlenspiel, das solange fortgesetzt wird, bis der von Ihnen geschätzte Wert mit der Zahl übereinstimmt, die das Cmdlet `Get-Random` generiert hat.</span><span class="sxs-lookup"><span data-stu-id="66114-134">The previous example is a numbers game that continues until the value you guess equals the same number that the `Get-Random` cmdlet generated.</span></span>

<span data-ttu-id="66114-135">`Do While` ist genau das Gegenteil.</span><span class="sxs-lookup"><span data-stu-id="66114-135">`Do While` is just the opposite.</span></span> <span data-ttu-id="66114-136">Sie wird ausgeführt, solange die angegebene Bedingung als „true“ ausgewertet wird.</span><span class="sxs-lookup"><span data-stu-id="66114-136">It runs as long as the specified condition evaluates to true.</span></span>

```powershell
$number = Get-Random -Minimum 1 -Maximum 10
do {
  $guess = Read-Host -Prompt "What's your guess?"
  if ($guess -lt $number) {
    Write-Output 'Too low!'
  } elseif ($guess -gt $number) {
    Write-Output 'Too high!'
  }
}
while ($guess -ne $number)
```

```Output
What's your guess?: 1
Too low!
What's your guess?: 2
Too low!
What's your guess?: 3
Too low!
What's your guess?: 4
```

<span data-ttu-id="66114-137">Dieselben Ergebnisse werden mit einer `Do While`-Schleife erzielt, indem die Testbedingung in „ungleich“ umgekehrt wird.</span><span class="sxs-lookup"><span data-stu-id="66114-137">The same results are achieved with a `Do While` loop by reversing the test condition to not equals.</span></span>

<span data-ttu-id="66114-138">`Do`-Schleifen werden immer mindestens einmal ausgeführt, weil die Bedingung am Ende der Schleife ausgewertet wird.</span><span class="sxs-lookup"><span data-stu-id="66114-138">`Do` loops always run at least once because the condition is evaluated at the end of the loop.</span></span>

### <a name="while"></a><span data-ttu-id="66114-139">While</span><span class="sxs-lookup"><span data-stu-id="66114-139">While</span></span>

<span data-ttu-id="66114-140">Ähnlich wie bei der `Do While`-Schleife, wird eine `While`-Schleife ausgeführt, solange die angegebene Bedingung erfüllt (true) ist.</span><span class="sxs-lookup"><span data-stu-id="66114-140">Similar to the `Do While` loop, a `While` loop runs as long as the specified condition is true.</span></span> <span data-ttu-id="66114-141">Der Unterschied besteht jedoch darin, dass eine `While`-Schleife die Bedingung am Anfang der Schleife auswertet, bevor überhaupt Code ausgeführt wird.</span><span class="sxs-lookup"><span data-stu-id="66114-141">The difference however, is that a `While` loop evaluates the condition at the top of the loop before any code is run.</span></span> <span data-ttu-id="66114-142">Somit wird sie gar nicht ausgeführt, wenn die Bedingung als „false“ ausgewertet wird.</span><span class="sxs-lookup"><span data-stu-id="66114-142">So it doesn't run if the condition evaluates to false.</span></span>

```powershell

  $date = $date.AddDays(1)
}
Write-Output $date
```

```Output
Thursday, November 23, 2017 12:00:00 AM
```

<span data-ttu-id="66114-143">Im vorherigen Beispiel wird berechnet, an welchem Tag in den USA „Thanksgiving“ (Erntedankfest) ist.</span><span class="sxs-lookup"><span data-stu-id="66114-143">The previous example calculates what day Thanksgiving Day is on in the United States.</span></span> <span data-ttu-id="66114-144">Dies ist immer am vierten Donnerstag im November.</span><span class="sxs-lookup"><span data-stu-id="66114-144">It's always on the fourth Thursday of November.</span></span> <span data-ttu-id="66114-145">Die Schleife beginnt also mit dem 22. Tag im November und fügt einen Tag hinzu, solange der Wochentag nicht gleich Donnerstag ist.</span><span class="sxs-lookup"><span data-stu-id="66114-145">So the loop starts with the 22nd day of November and adds a day while the day of the week isn't equal to Thursday.</span></span> <span data-ttu-id="66114-146">Wenn der 22. ein Donnerstag ist, wird die Schleife überhaupt nicht ausgeführt.</span><span class="sxs-lookup"><span data-stu-id="66114-146">If the 22nd is a Thursday, the loop doesn't run at all.</span></span>

## <a name="break-continue-and-return"></a><span data-ttu-id="66114-147">Break, Continue und Return</span><span class="sxs-lookup"><span data-stu-id="66114-147">Break, Continue, and Return</span></span>

<span data-ttu-id="66114-148">`Break` dient dem Unterbrechen einer Schleife.</span><span class="sxs-lookup"><span data-stu-id="66114-148">`Break` is designed to break out of a loop.</span></span> <span data-ttu-id="66114-149">Es wird auch häufig mit der `switch`-Anweisung verwendet.</span><span class="sxs-lookup"><span data-stu-id="66114-149">It's also commonly used with the `switch` statement.</span></span>

```powershell
for ($i = 1; $i -lt 5; $i++) {
  Write-Output "Sleeping for $i seconds"
  Start-Sleep -Seconds $i
  break
}
```

```Output
Sleeping for 1 seconds
```

<span data-ttu-id="66114-150">Die im vorherigen Beispiel gezeigte `break`-Anweisung bewirkt, dass die Schleife bei der ersten Iterationen beendet wird.</span><span class="sxs-lookup"><span data-stu-id="66114-150">The `break` statement shown in the previous example causes the loop to exit on the first iteration.</span></span>

<span data-ttu-id="66114-151">„Continue“ ist so konzipiert, dass zur nächsten Iteration einer Schleife gesprungen wird.</span><span class="sxs-lookup"><span data-stu-id="66114-151">Continue is designed to skip to the next iteration of a loop.</span></span>

```powershell

  if ($i -eq 3) {
    continue
  }
  Write-Output $i
}
```

```Output

2
4
5
```

<span data-ttu-id="66114-152">Das vorherige Beispiel gibt die Zahlen 1, 2, 4 und 5 aus.</span><span class="sxs-lookup"><span data-stu-id="66114-152">The previous example will output the numbers 1, 2, 4, and 5.</span></span> <span data-ttu-id="66114-153">Es überspringt die Zahl 3 und fährt mit der nächsten Iteration der Schleife fort.</span><span class="sxs-lookup"><span data-stu-id="66114-153">It skips number 3 and continues with the next iteration of the loop.</span></span> <span data-ttu-id="66114-154">Ähnlich wie bei `break`, unterbricht `continue` die Schleife, wenn auch nur für die aktuelle Iteration.</span><span class="sxs-lookup"><span data-stu-id="66114-154">Similar to `break`, `continue` breaks out of the loop except only for the current iteration.</span></span> <span data-ttu-id="66114-155">Die Ausführung wird mit der nächsten Iteration fortgesetzt, anstatt die Schleife zu unterbrechen und zu beenden.</span><span class="sxs-lookup"><span data-stu-id="66114-155">Execution continues with the next iteration instead of breaking out of the loop and stopping.</span></span>

<span data-ttu-id="66114-156">„Return“ ist so konzipiert, dass der vorhandene Bereich verlassen wird.</span><span class="sxs-lookup"><span data-stu-id="66114-156">Return is designed to exit out of the existing scope.</span></span>

```powershell

  if ($n -ge 4) {
    Return $n
  }
}
```

```Output

```

<span data-ttu-id="66114-157">Beachten Sie, dass im vorherigen Beispiel „return“ das erste Ergebnis ausgibt und dann die Schleife beendet.</span><span class="sxs-lookup"><span data-stu-id="66114-157">Notice that in the previous example, return outputs the first result and then exists out of the loop.</span></span> <span data-ttu-id="66114-158">Eine gründlichere Erklärung der result-Anweisung finden Sie in einem meiner Blogartikel: [„Das PowerShell-Schlüsselwort ‚return‘“][].</span><span class="sxs-lookup"><span data-stu-id="66114-158">A more thorough explanation of the result statement can be found in one of my blog articles: ["The PowerShell return keyword"][].</span></span>
